# 📌 🦇 Gotham Intel: Agency Management System

Gotham Intel — это защищенная информационная система, разработанная для Брюса Уэйна и Альфреда Пенниуорта. Она предназначена для учета агентов, работающих в Готэме, мониторинга их статусов и ведения досье.

Проект построен на базе Flask с использованием современной архитектуры (Blueprints, Application Factory) и строгим логированием всех действий.

---

## ✏️ 🛠 Технологический стек
- Backend: Python 3.12+, Flask
- Database: SQLAlchemy (SQLite)
- Testing: Pytest
- Architecture: Blueprints & Application Factory Pattern
- Logging: Кастомный модуль логирования действий в системе

---

## ✏️ 🚀 Функционал
- Управление агентами: Добавление, редактирование, просмотр и удаление (CRUD).
- Система безопасности: Использование секретных ключей доступа.
- Логирование: Каждое действие (создание, изменение, удаление) записывается в лог-файл для аудита.
- Обработка ошибок: Кастомные страницы для ошибок 404 и 500 в стиле Готэма.
- Автоматическое тестирование: Полное покрытие тестами основных маршрутов и логики БД.

## 📝 Система логирования

Приложение реализует комплексную систему логирования, которая:

- Записывает все события приложения и ошибки.
- Использует различные уровни логирования (INFO, WARNING, ERROR, DEBUG, CRITICAL).
- Обеспечивает цветной вывод в консоль для лучшей читаемости.
- Хранит логи в отдельных файлах(app.log для общей информации, error.log для ошибок) в общей директории `logs/`.

### Файлы логов

- **app.log**: Содержит общую информацию о работе приложения.
- **error.log**: Записывает все сообщения уровня `ERROR` и выше.

### Компоненты логирования

Настройка системы логирования осуществляется через следующие файлы:
- **my_logger_config.py**: Настраивает конфигурацию логирования
- **my_logging_config.yaml**: Конфигурация для различных логгеров
- **my_color_formatter.py**: Кастомный форматтер для цветного вывода в консоль

## 🧪 Тестирование

Для запуска тестов используется `pytest`. Мы проверяем создание агентов, редиректы, обработку ошибок и работу базы данных.

### Запуск тестов

Тесты можно запустить из командной строки:
- `pytest` - Краткий вариант результатов тестирования
- `pytest -v` - Более детальный вывод результатов тестирования

Также файл с тестированием можно запустить как и обычный код, через кнопку `run`.

### Примеры тестов

1. Проверяет успешное удаление досье агента.
```python
def test_delete_agent(client, sample_agent_id):
    """
    Тестирует процесс уничтожения досье агента.
    Проверяет:
    1. Появление подтверждающего сообщения ("Досье уничтожено").
    2. Отсутствие записи в базе данных после удаления.
    """
    response = client.get(f'/delete/{sample_agent_id}', follow_redirects=True)
    assert response.status_code == 200
    assert "Досье уничтожено" in response.get_data(as_text=True)
    assert Agent.query.get(sample_agent_id) is None
```
